    def knapsack_backtracking(weights, values, capacity):
        """
        Solves the knapsack problem using backtracking.
    
        :param weights: A list of item weights.
        :param values: A list of item values.
        :param capacity: The maximum capacity of the knapsack.
        :return: The maximum value that can be obtained.
        """
        n = len(weights)
        max_val = 0
    
        def backtrack(i, curr_weight, curr_val):
            nonlocal max_val
            if i == n or curr_weight == capacity:
                if curr_val > max_val:
                    max_val = curr_val
                return
            if curr_weight + weights[i] <= capacity:
                backtrack(i + 1, curr_weight + weights[i], curr_val + values[i])
            backtrack(i + 1, curr_weight, curr_val)
    
        backtrack(0, 0, 0)
        return max_val

Knapsack problem

If I have a knapsack of capacity c and n items with their weights and values given, how can I fill the knapsack to maximize the total value of the items inside without exceeding the capacity?

What would the algorithm do without backtracking?

Without backtracking, a naive approach to solve the Knapsack problem would be to generate all possible subsets of the items and calculate the total value and weight of each subset. The algorithm would then compare the total weight of each subset with the capacity of the knapsack to determine if it is feasible. Among the feasible subsets, the algorithm would then select the one with the highest total value as the optimal solution.

However, this approach would be inefficient as the number of possible subsets increases exponentially with the number of items, making it impractical for large instances of the Knapsack problem.

Introducing backtracking

A backtracking algorithm is a way of computing a solution to a problem when constraints are known.

Backtracking can be used to solve the Knapsack problem by trying out all possible combinations of items to include in the knapsack, and backtracking when the weight limit is exceeded or all items have been considered.

This approach avoids the need to check every single subset of items, which would be inefficient. Instead, it only considers combinations of items that are valid up to a certain point, and backtracks when necessary.
